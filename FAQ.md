# Часто задаваемые вопросы (FAQ)

## 🚀 Запуск и настройка

### Как получить BOT_TOKEN?
1. Откройте Telegram
2. Найдите [@BotFather](https://t.me/BotFather)
3. Отправьте `/newbot`
4. Следуйте инструкциям
5. Скопируйте токен и добавьте в `.env`

### Бот не запускается
**Проверьте:**
- Python версии 3.11 или выше: `python --version`
- BOT_TOKEN правильно указан в `.env`
- Все зависимости установлены: `pip install -r requirements.txt`
- Логи в файле `bot.log`

### Ошибка "Invalid token"
- Проверьте что токен скопирован полностью
- Убедитесь что нет лишних пробелов
- Токен должен выглядеть так: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

## 👥 Пользователи и база данных

### Где хранятся данные?
По умолчанию в SQLite файле `data/bot.db`

### Как перейти на PostgreSQL?
1. Раскомментируйте секцию postgres в `docker-compose.yml`
2. Измените `DATABASE_URL` в `.env`
3. Перезапустите бота

### Как сделать бэкап БД?
```bash
# SQLite
cp data/bot.db data/bot.db.backup

# PostgreSQL
docker-compose exec postgres pg_dump -U botuser botdb > backup.sql
```

### Как восстановить из бэкапа?
```bash
# SQLite
cp data/bot.db.backup data/bot.db

# PostgreSQL
docker-compose exec -T postgres psql -U botuser botdb < backup.sql
```

## 🔧 Технические вопросы

### Какой хостинг выбрать?
**Бесплатные:**
- Oracle Cloud (Always Free)
- Railway.app (500 часов/месяц)
- Render.com (750 часов/месяц)

**Платные:**
- DigitalOcean ($5/месяц)
- Hetzner ($3/месяц)
- AWS/GCP (от $5/месяц)

### Сколько пользователей выдержит?
- SQLite: до 1000 активных пользователей
- PostgreSQL: до 100,000+ пользователей
- С Redis и масштабированием: миллионы

### Как масштабировать бота?
1. Перейдите на PostgreSQL
2. Добавьте Redis для кэширования
3. Используйте webhook вместо polling
4. Запустите несколько инстансов бота
5. Используйте load balancer (nginx)

### Webhook или polling?
- **Polling** (по умолчанию): проще настроить, подходит для начала
- **Webhook**: быстрее, эффективнее, для production

## 🐛 Решение проблем

### Бот не отвечает на сообщения
1. Проверьте что бот запущен: `docker-compose ps`
2. Посмотрите логи: `docker-compose logs -f bot`
3. Проверьте интернет-соединение
4. Убедитесь что токен валидный

### Ошибка "Database is locked"
- SQLite не поддерживает много одновременных записей
- Решение: перейдите на PostgreSQL

### Рассылка не работает
1. Проверьте что планировщик запущен
2. Посмотрите логи
3. Убедитесь что время настроено правильно (UTC)

### Мемы не генерируются
1. Проверьте что Pillow установлен: `pip install Pillow`
2. Убедитесь что есть права на запись в `/tmp`
3. Проверьте логи на ошибки
