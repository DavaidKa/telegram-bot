# Changelog

Все значимые изменения в проекте будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.0] - 2024-03-13

### ✨ Добавлено
- Базовая структура проекта
- Регистрация и авторизация пользователей
- Генератор мемов с несколькими шаблонами
- Система викторин (кино, музыка, география, наука)
- Мини-игры (кости, камень-ножницы-бумага, предсказания)
- Система уровней и опыта
- Реферальная программа с бонусами
- VIP подписка ($2/месяц)
- Интеграция с платежными системами (PayPal, YooMoney, QIWI)
- Админ-панель с статистикой
- Система рассылок
- Ежедневные уведомления
- Таблица лидеров
- Защита от спама (throttling middleware)
- Логирование всех действий
- Docker поддержка
- Документация (README, FAQ, DEPLOYMENT, etc.)

### 🗄️ База данных
- Модель User (пользователи)
- Модель Payment (платежи)
- Модель GeneratedContent (созданный контент)
- Модель Quiz (викторины)
- Модель QuizResult (результаты викторин)
- Модель Achievement (достижения)
- SQLite по умолчанию
- Поддержка PostgreSQL

### 🎨 Функции
- Создание мемов с текстом
- Мотивационные цитаты
- Случайные мемы
- Викторины по категориям
- Игра в кости
- Камень-ножницы-бумага
- Ежедневные предсказания
- Система опыта и уровней
- Реферальные бонусы

### 👨‍💼 Админ функции
- Просмотр статистики
- Рассылка сообщений
- Список пользователей
- Выдача VIP вручную
- Полная статистика

### 📚 Документация
- README.md - основная документация
- QUICKSTART.md - быстрый старт
- DEPLOYMENT.md - инструкции по развертыванию
- GROWTH_STRATEGY.md - стратегия роста
- IMPROVEMENTS.md - идеи улучшений
- FAQ.md - частые вопросы
- PROJECT_INFO.md - информация о проекте
- CHANGELOG.md - история изменений

### 🐳 DevOps
- Dockerfile для контейнеризации
- docker-compose.yml для оркестрации
- Скрипты запуска (start.sh, start.bat)
- .gitignore для Git
- .env.example для конфигурации

## Версионирование

Мы используем [SemVer](http://semver.org/) для версионирования:

- **MAJOR** версия - несовместимые изменения API
- **MINOR** версия - новый функционал с обратной совместимостью
- **PATCH** версия - исправления багов с обратной совместимостью
